hibernate入门

发表于 2019-03-06 | 分类于 web基础 | 阅读次数:

简介

struts(架子) spring(春天) hibernate(冬眠,数据冬眠)

hibernate: 是一款ORM (Object Relative Mapping) 框架,代替JDBC和DButils帮助我们完成对数据的操作.属于持久层.

好处: 实现了ORM思想,将关系数据库中的表数据映射成对象,把对数据库的操作转化为对对象的操作.

基本步骤: 导包(request+驱动包) + 准备实体类,配置orm元数据(映射文件) + 创建主配置文件 + 书写代码

结构:

实体类创建规则

配置orm元数据(映射文件)

映射文件：反映持久化类和数据库表的映射信息。即配置实体类映射到数据库中的哪个表，以及类中的哪个属性对应表中的哪个字段。

主配置

hibernate主配置：配置数据库连接以及Hibernate运行时所需要的各个属性的值。文件名：hibernate.hbm.xml

API

Hibernate对象状态：瞬时状态、持久化状态、游离|托管状态（只有第一个无唯一标志oid，只有第二个与session关联）

状态 是否在session缓存中 是否与session实例相关联 在数据中是否有与之对应的记录
Transient(短暂) 否 否 否
Persistent(固执) 是 是 是
Detached(分离) 否 否 是

其中，持久化特点：持久化对象的任何变化，都会在事务提交时，自动同步到数据库中

一对多&多对一关系的操作

查询

HQL(不复杂时可多表查询)：hibernate独家查询语言。

  • 优点：可读性好，功能强大效率高。
  • 缺点：由于是字符串形式，只有在运行时才被解析，故扩展性差。

Criteria(单表查询)：hibernate自创的无语句面向对象查询

  • QBC优点：提供面向对象的接口，编译时即可被解析，便于调试，扩展性好，允许用户扩展Criteria接口。
  • QBC缺点：可读性差；不支持报表查询和子查询。

原生SQL查询(复杂的业务查询)

知识补充

一、orm(对象关系映射)框架分为4级：

  • hibernate：属于4级，完全面向对象操作数据库
  • mybatis：属于2级
  • dbutils：属于1级

二、Hibernate一级缓存：session中有一个可存放对象信息的集合。减少对数据库访问次数，提高操作数据库的效率。

​ a、提高查询效率：首次查询会将对象放入一级缓存，再次查询时直接会返回缓存中的，而不是查询数据库

​ b、减少不必要的修改：会使用快照对比修改前后的属性区别，只执行一次修改。

三、对HibernateDaoSupport，ActionSupport等类中所指的Support的理解**：都用到了IoC和工厂模式，即依赖注入，将程序代码中依赖对象的管理交由相应的容器去管理，如在配置文件中注入配置。
